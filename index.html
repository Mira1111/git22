Загальна інформація

Роботу над сайтами поділяють на :

1. Дизайн - підбір кольорової гами сайту, створення та обробка зображень для розташування на сайті.
2. Верстку - точне відтворення на сайті всього згідно макету замовника сайту, дизайн сайту. Забезпечує, щоб сайт виглядав однаково у всіх браузерах і всіх операційних системах та при любому розмірі екрана було зручно переглядати сайт (адаптивна верстка), та у випадках зміни налаштування розмірів шрифтів браузера чи при відключенні перегляду малюнків.
3. Front-end - забезпечення клієнтської сторони сайту, тобто роботи всіх скриптів, забезпечення виконання всіх функцій сайту (перехід між сторінками, робота кнопок і т. д. ). Тобто, front-end розробник (front-end developer) створює логістику (логіку) сайту. Крім того, front-end developer має знати також і верстку. Тут потрібні знання HTML, CSS, JavaScript (JS).
4. Back-end - забезпечення роботи серверної сторони сайту, тобто звернення до бази даних, взяття інформації з бази і відображення цієї інформації на сайті, пошук та зберігання інформації в ній іншими словами:
front-end - це дизайн, верстка, наповнення сайту та навігація (рух) по сайту.
Full-stack – всі 4 пункти разом.

Види сайтів:

1. Cтатичні - не має back-end. Створюється через HTML, CSS, інколи JS
2. Динамічні - має back-end. Створюється через HTML, CSS, обов'язково JS та додаткові мови програмування типу PHP, Ruby. НТМL (Hyper Text Markup Language) - мова розмітки гіпертексту (веб-сторінки). Цю мову створив 1986-1991 роках англієць Тім Бернерс-Лі. Версії НТМL: НТМL , НТМL +, НТМL 2, НТМL 3, НТМL3.2, НТМL 4, НТМL 4.01, XHTML, XНТМL1.1, XНТМL2, НТМL 5

Веб-редактори:

Створювати веб-сторінки можна використовуючи наступні програми, які відносяться до категорії редактори коду:
1. Оnline-редактори - для написання коду зразу в браузері. Приклад: JSFiddle.net, LiveWeave.com
2. Оffline-редактори - частіше використовують для написання великих проектів. Їх ділять на прості редактори та смарт-редактори. В цих редакторах код підсвічується різними кольорами і є можливість автозаповнення команд коду. Приклад простих редакторів: Блокнот, Notepad++. Приклад смартредакторів: Sublime (https://www.sublimetext.com), Brackets, Atom
3. інтегроване середовище - має найбільше функцій і є найкращим для використання, його мінус - сильно загружає систему. Приклад: Web-Storm, Visual Studio Найпростіший спосіб створення HTML-документів:
1. Відкриваємо програму Блокнот
2. Набираємо у редакторі Блокнот html-код веб-сторінки
3. Зберігаємо файл як HTML-документ (вибираємо команду "Сохранить как", тип файлу вибираємо "Все файлы", а ім'я файлу записуємо із розширенням .html).
Відкривається такий HTML-документ для перегляду через браузер, при чому в різних браузерах деякі теги можуть дещо по іншому працювати. *.html-файл може бути одночасно відкритий і у редакторі Блокнот, і в браузері.
Основну сторінку сайту зазвичай зберігають під назвою index При редагуванні HTML-документа зручно одразу переглядати результат редагування, для цього потрібно в веб-редакторі зберігати файл командою "Сохранить" (або Ctrl+S), розгортати вікно браузера в якому відкрита веб-сторінка і натискати кнопку "Обновить" (або клавішу F5). На екрані одразу відображатимуться зміни на веб-сторінці.
Для читабельності коду в HTML додають пропуски, в основному це Enter та Tab, причому Enter ставлять в кінці кожного рядка, а Tab ставлять для відображення вкладення одного тегу в інший. Створення html-документа в Sublime та збереження :
1. File-New file
2. File-Save with Encoding-UTF-8, записати назву файлу і вибрати тип файлу HTML(*.html....)
